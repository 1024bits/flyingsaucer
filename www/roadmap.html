Flying Saucer Roadmap

Abstract: This roadmap is designed to take us from where we are now, a slow partial implementation of CSS with no real use, to a fast completely CSS 2.1 compliant renderer with demonstration applications and a viable userbase. This document is comprised of six releases each with a distinct focus and timetable. These are the definitive milestones of Flying Saucer that will drive development and ensure we make real progress

Releases:

  1. Initial release to Java.net
  2. First progress release (7/14/04)
  3. Cleanup release
  4. Major infrastructure planning

Release 3 : Cleanup

Purpose: to clean up the code base, fix a few outstanding bugs, and get it into shape for some major infrastructure changes. Also to show off the new AboutBox demo and hopefully get some developer interest. This release needs to be good enough to get us through the long cycle of release 4.

Timetable: 1 month (8/31?)

  1. html: test jtidy add on
  2. html: good enough for javadocs
  3. tables: finish standard table layout
  4. tables: finish table styling
  5. forms: all form elements
  6. forms: programmatic access
  7. forms: get and post support
  8. browser: add get title to the htmlpanel
  9. browser: splashpage on browser
  10. browser: fix webstart security issues. load demos via web
  11. browser: reload
  12. apps: javadoc viewer
  13. apps: quickpanel (stuff html url and go)
  14. doc: getting started tutorial
  15. doc: links to good css references
  16. bugs: fix show stopper bugs (inft loop prob)

Release 4: Rearchitecture

Purpose: This release will have no new features. It's entire purpose is to refactor the codebase, design proper interfaces to each module, and put in solid unit/regression/speed tests. This will require a CVS branch, probably, since we want the main branch (from v3) to still be able to take bug fixes. We will also take this time to drum up more publicity.

Timetable: 2 months (10/31?)

  1. css: new modular infrastructure
  2. css: new XR style objects
  3. css: configurable default/useragent/user/author css
  4. testing: headless unit tests for current supported properties
  5. testing: auto generates compliance report
  6. testing: speed report logging: comprehensive configurable logging
  7. browser: resource caching layer
  8. browser: network access layer (jakarta http get?)
  9. doc: javadocs
  10. publicity: solicit other projects
  11. publicity: interviews
  12. publicity: press releases

Release 5: Compliance and Advanced Features

Date: 1 month (11/31?)

Purpose: build on the existing infrastructure to finish up the low hanging fruit in terms of missing css features.

  1. css: @media selection
  2. css: @import linking to style sheets
  3. layout: proper list bullet positioning
  4. rendering: support other border types
  5. browser: ssl support
  6. browser: full mouse selection w/ styled copy and paste
  7. browser: mouse selection w/ plain text copy and paste
  8. apps: docbook viewer via XSLT
  9. browser: inline directory generation (make it pretty)
  10. doc: faq

Release 6: Compliance and Advanced Features

Timetable: 2 months

Purpose: full compliance for positioning and partial reflow. finally implement the old non-incremental table layout algorithm, and do more internal refactoring.

  1. layout: partial dynamic reflow for :hover
  2. rendering: partial repainting (dirty bits)
  3. tables: old netscape style table algorithm
  4. layout: compliant CSS positioning (absolute, fixed, relative)
  5. layout: compliant CSS float positioning
  6. html: good enough for standard docbooks
  7. html: good enough for yahoo and amazon
  8. layout: factor out hardcoded element names. just use layout property
  9. apps: docbook viewer via direct CSS
  10. doc: embedding/configuration docs for each subsystem
  11. doc: XML rendering
